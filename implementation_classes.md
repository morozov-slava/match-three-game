На первый взгляд кажется, что реализация игры возможна с использованием базовых структур данных в python (в основном типов `list` и `dict`).

Основные вычислительные процессы, где имеет смысл рассматривать возможность добавления классов реализации - это работа с игровым полем (`PlayingField`).
На уровне игрового поля нам потребуются следующие операции:
- Генерация игрового поля 
- Поиск комбинаций
- Смещение элементов сверху-внизу после удаления комбинации с поля.
- Перестановка элементов местами

В качестве оптимальной структуры данных видится использование двумерного массива (`2DArray`) для хранения элементов.

В таком, случае, соответственно:
- **Перестановка элементов местами** будет осуществляться за `O(1)`, т.к. обращение будет по индексам (см. маппинг пользовательский ввод -> индексы ниже).
- **Смещение элементов на один уровень сверху-вниз** также будет осуществляться за `O(1)`, т.к. будет манипуляция на уровне индексов.
- Поиск комбинаций будет осуществляться за `O(N x N)`,
  где `N` - количество строк/столбцов (в нашем случае это квадратная матрица).
  В целом, это линейная скорость, что при малом размере поля (очевидно, что размер поля для игры не будет величиной, которая будет существенно расти в будущем, поэтому данная временная сложность нас устраивает).
  
  Важным преимуществом данного подхода является:
	- Возможность масштабирования под диагональные и более сложные формы комбинаций (N в ряд).
	- Отсутствие зависимости от формы поля.
	  (на случай, если захотим сделать, например, прямоугольное поле)

Для генерации игрового поля отдельные структуры данных нам не нужны и это можно реализовать также на уровне двумерного массива.

При пользовательском вводе запроса на перестановку элементов необходимо будет соотносить названия клеток с координатами в двумерном массиве.
Для этого оптимально использовать хэш-таблицу (`dict` в Python).

Пример справочника:

```python
{
 "A1": (0, 0), 
 "A2": (1, 0),
 ...
 "B1": (0, 1),
 "B2": (1, 1),
 ...
}

# где 1-й элемент - индекс строки, а 2-й элемент - индекс столбца в 2D Array
```
