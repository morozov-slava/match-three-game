**Задание:**

Формируем архитектуру, задаём принципы поведения системы: схемы создания объектов, обработки событий, увязки с выбранными фреймворками и технологиями, и тесты для типовых сценариев (проектирование). 


### 1. Архитектура игры


#### 1.1. Общий подход к архитектуре

Архитектура игры формируется на базе ООАП подхода с использованием модульной разработки и АТД в качестве основных структур данных в проекте.
В качестве архитектурного шаблона воспользуемся **MVC** со следующим разделение логики на компоненты:

- **Model** - игровая логика и структура данных (сетка, очки, история ходов).
- **View** - отображение и ввода данных через консоль.
- **Controller** - обработка пользовательского ввода, управление взаимодействием между игровой логикой и отображением.


#### 1.2. Основные АТД

`PlayingField` - объект игрового поля
`ElementGenerator` - генерация элементов для игрового поля
`PlayingFieldGenerator` - генератор игрового поля для конкретной игровой сессии
`CombinationsChecker` - объект для проверки комбинаций на игровом поле
`StatisticsMetric` - объект для определения метрик для подсчёта статистики
`GameStatisticsCounter` - объект для подсчёта метрик игровой статистики
`GameStatisticsDisplayer` - объект для отображения метрик игровой статистики
`Bonus` - объект для хранения информации о некотором игровом бонусе
`BonusesChecker` - объект для проверки наличия бонусов после совершения некоторого хода
`InputCommandsHandler` - объект для обработки команд игрока
`InputCommandsRunner` - объект для исполнения команд игрока
`Game` - объект для запуска и управления игрой


### 2. Принципы поведения системы


#### 2.1. Создание объектов

- При запуске `Game` создаёт `PlayingField` размером 8x8 с помощью `PlayingFieldGenerator` с заполненными значениями.
- Инициализация игрового поля в начале игры должна исключать наличие комбинаций.
- Создаётся объект `GameStatisticsCounter` и начинается сбор статистики.
- Создаётся объект `BonusChecker` и начинается проверка состояния `PlayingField` после каждого хода.


#### 2.2. Обработка событий

- Игрок вводит координаты (например, "C2 D2").
- `InputCommandsHandler` проверяет валидность команды.
- Далее `InputCommandsRunner` отвечает за исполнение команды (если проблем с командой нет).
- Далее `PlayingField` производит перестановку значения.
- `CombinationsChecker` находит все комбинации на игровом поле.
- `BonusesChecker` добавляет бонусы (если применимо в рамках игровой сессии).
- `PlayingField` удаляет элементы в рамках комбинации.
- `ElementGenerator` генерирует элементы для игрового поля на место удалённых.
- Цикл генерации продолжается, пока есть новые совпадения.
- Когда на игровом поле отсутствуют совпадения — игроку предоставляется следующая возможность хода.
- Игра завершается, если `PlayingField` не находит возможных ходов.
